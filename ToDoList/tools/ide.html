IDE 开发环境全面手册

VS Code 基础配置

Visual Studio Code 是微软开发的免费代码编辑器。

// settings.json 配置
{
  "editor.fontSize": 14,
  "editor.tabSize": 2,
  "editor.insertSpaces": true,
  "editor.wordWrap": "on",
  "editor.minimap.enabled": false,
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "workbench.colorTheme": "One Dark Pro",
  "terminal.integrated.fontSize": 12,
  "git.enableSmartCommit": true,
  "git.confirmSync": false
}
实际场景：个性化配置、提高开发效率、团队统一设置。
常见错误：配置冲突、插件兼容性问题。

Vim/Neovim 基础操作

// Vim 基础命令

v           // 进入可视模式

移动命令:
h, j, k, l  // 左下上右
w, b        // 单词前进/后退

dd          // 删除行
yy          // 复制行
p           // 粘贴
u           // 撤销
Ctrl + r    // 重做
实际场景：服务器编辑、快速修改、轻量级编辑。

Vim 配置文件

通过 .vimrc 配置文件自定义 Vim。

" .vimrc 配置示例
set number              " 显示行号
set relativenumber      " 相对行号
set tabstop=4          " Tab 宽度
set shiftwidth=4       " 缩进宽度
set expandtab          " 使用空格代替 Tab
set autoindent         " 自动缩进
set smartindent        " 智能缩进
set hlsearch           " 高亮搜索
set incsearch          " 增量搜索
set ignorecase         " 忽略大小写
set smartcase          " 智能大小写
set wrap               " 自动换行
set ruler              " 显示光标位置
set showcmd            " 显示命令
set wildmenu           " 命令补全菜单

" 键位映射
let mapleader = ","
nnoremap w :w
nnoremap q :q
实际场景：个性化配置、提高效率、团队标准。

容器开发

使用容器技术创建一致的开发环境。

# Dockerfile 开发环境
FROM node:16-alpine

WORKDIR /app

# 安装开发工具
RUN apk add --no-cache \
    git \
    curl \
    vim \
    zsh

# 安装 Oh My Zsh
RUN sh -c "$(curl -fsSL https://raw.github.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# 复制配置文件
COPY .vimrc /root/.vimrc
COPY .zshrc /root/.zshrc

# 安装全局包
RUN npm install -g \
    @vue/cli \
    create-react-app \
    typescript \
    nodemon

EXPOSE 3000 8080
CMD ["zsh"]
实际场景：环境一致性、快速搭建、团队协作。

远程开发

通过远程开发提高开发灵活性。

# VS Code Remote 配置
# .vscode/settings.json
{
  "remote.SSH.remotePlatform": {
    "myserver": "linux"
  },
  "remote.SSH.configFile": "~/.ssh/config"
}

# SSH 配置 ~/.ssh/config
Host myserver
    HostName 192.168.1.100
    User developer
    Port 22
    IdentityFile ~/.ssh/id_rsa
    ForwardAgent yes

# Docker 开发容器
# .devcontainer/devcontainer.json
{
  "name": "Node.js Development",
  "image": "node:16",
  "features": {
    "ghcr.io/devcontainers/features/git:1": {}
  },
  "customizations": {
    "vscode": {
      "extensions": [
        "ms-vscode.vscode-typescript-next"
      ]
    }
  }
}
实际场景：云端开发、资源共享、环境隔离。

最佳实践

  • 选择适合项目类型的 IDE
  • 统一团队开发环境配置
  • 合理使用插件，避免过度依赖
  • 掌握快捷键提高效率
  • 定期备份配置文件
  • 使用版本控制管理配置
  • 关注性能，避免卡顿

常见问题

  • Q: IDE 运行缓慢如何优化？
    A: 增加内存分配、禁用不必要插件、清理缓存。
  • Q: 如何同步多设备配置？
    A: 使用云同步、Git 仓库、配置文件管理。
  • Q: 插件冲突如何解决？
    A: 逐个禁用测试、查看日志、更新版本。
