Git 全面手册

初始化仓库

Git 是分布式版本控制系统，用于跟踪文件变化和协作开发。

# 初始化新仓库
git init

# 配置用户信息
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 查看配置
git config --list
实际场景：项目开始、团队协作、版本管理。
常见错误：忘记配置用户信息导致提交失败。

添加与提交

Git 的基本工作流程是添加文件到暂存区，然后提交到仓库。

# 添加文件到暂存区
git add file.txt
git add .  # 添加所有文件

# 提交更改
git commit -m "Add new feature"

# 添加并提交（跳过暂存区）
git commit -am "Update existing file"

# 修改最后一次提交
git commit --amend -m "Updated commit message"
实际场景：代码开发、功能实现、bug 修复。
常见错误：提交信息不清晰、提交粒度过大。

最佳实践

  • 提交信息要清晰描述变更内容
  • 每次提交应该是一个逻辑单元
  • 使用约定式提交格式

分支基础

分支是 Git 的核心功能，允许并行开发不同功能。

# 查看分支
git branch
git branch -a  # 查看所有分支

# 创建分支
git branch feature/new-feature

# 切换分支
git checkout feature/new-feature
git switch feature/new-feature  # Git 2.23+

# 创建并切换分支
git checkout -b feature/new-feature
git switch -c feature/new-feature  # Git 2.23+

# 删除分支
git branch -d feature/new-feature
git branch -D feature/new-feature  # 强制删除
实际场景：功能开发、实验性代码、并行工作。
常见错误：在错误的分支上开发、忘记切换分支。

合并分支

将不同分支的更改合并到主分支。

# 合并分支
git checkout main
git merge feature/new-feature

# 无快进合并（保留分支历史）
git merge --no-ff feature/new-feature

# 压缩合并（将多个提交合并为一个）
git merge --squash feature/new-feature

# 取消合并
git merge --abort
实际场景：功能完成、代码集成、发布准备。
常见错误：合并冲突处理不当、丢失代码。
